# Repository Knowledge Map

This directory is the repo-local knowledge base for the steel BIM project. Keep `AGENTS.md` short and put durable rules here.

## Sections

- `architecture/` - data model and technical boundaries.
- `workflows/` - how Codex should operate in this repo.
- `quality/` - validation and checks.
- `decisions/` - architecture decision records.
- `exec-plans/` - active and completed execution plans.

## Harness Engineering Principles Used Here

- Humans steer; agents execute.
- Repository knowledge is the system of record.
- `AGENTS.md` is a map, not a full manual.
- Data shapes are parsed and validated at boundaries.
- Architecture rules should become mechanical checks when possible.
- Small, reviewable changes are preferred.
- If an agent struggles, improve the repo instructions, docs, scripts, or schemas.
